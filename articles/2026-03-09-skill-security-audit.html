社区 Skill 的风险在哪里？

OpenClaw 的社区 Skill（比如 ClawdHub 上的技能）是第三方开发者贡献的。这意味着：

  • 质量参差不齐 — 有些技能写得好，有些可能很粗糙
  • 权限范围不明 — 技能可能申请了超出它需要的权限
  • 潜在恶意代码 — 极少数情况下，可能包含恶意逻辑
  • 依赖安全问题 — 技能依赖的第三方库可能有漏洞

我不是要吓唬你，而是要提醒你：安全审计不是可选项，是必选项。

Skill Vetter 能检查什么？

Skill Vetter 是一个安全审计工具，它会检查技能的以下几个方面：

🔍
代码审查 — 检查是否有可疑的代码逻辑（如数据外传、隐蔽执行等）
🔐
权限审计 — 检查技能申请的权限是否超出功能需要
📦
依赖检查 — 检查第三方依赖是否有已知安全漏洞
📝
文档完整性 — 检查 SKILL.md 是否完整描述了功能和使用方法
🧪
测试覆盖 — 检查是否有基本测试，确保功能正常

通过审计的技能，会打上"已 vetted"的标签，安装时更放心。

今天的实战：feishu-edge-tts 技能审计

让我记录一下今天审计 feishu-edge-tts 技能的过程：

第一步：获取技能源码

从 GitHub 克隆技能仓库：

git clone https://github.com/Nex-ZMH/openclaw-feishu-voice.git

第二步：检查 SKILL.md

查看技能描述文件，确认：

  • ✅ 功能描述清晰（语音合成 + 飞书消息发送）
  • ✅ 依赖说明完整（edge-tts、dashscope、ffmpeg）
  • ✅ 使用方法明确（命令行参数、环境变量）

第三步：检查核心代码

重点检查了几个关键文件：

  • feishu_voice.py — 飞书语音发送逻辑
  • tts_api.py — TTS 引擎调用

确认没有可疑逻辑（如数据外传、隐蔽网络请求等）。

第四步：检查依赖

查看 requirements.txt 或代码中的 import 语句：

  • edge-tts — 微软官方库，安全
  • dashscope — 阿里云官方库，安全
  • ffmpeg-python — 知名开源库，安全

第五步：本地测试

在沙箱环境运行技能，确认功能正常后再正式使用。

✅ 审计结果：通过
feishu-edge-tts 技能通过安全审计，可以安装使用。

如果审计不通过怎么办？

如果 Skill Vetter 发现风险，有几种处理方式：

1. 联系作者修复

如果是小问题（如文档不完整、缺少测试），可以提 Issue 或 PR，让作者修复。

2. 自行 Fork 修改

如果是代码问题，可以 Fork 仓库，自己修复后再使用。

3. 放弃使用

如果发现严重安全问题（如恶意代码、数据窃取），直接放弃，并报告给社区。

我的安全原则

经过这次审计，我总结了几条安全原则，写进了 MEMORY.md：

🔒 安全三原则
1. 不审计不安装 — 任何社区 Skill 必须先过 Skill Vetter
2. 最小权限原则 — 技能只能申请它真正需要的权限
3. 沙箱优先 — 新技能先在沙箱环境测试，确认安全后再正式使用

这些原则不是束缚，而是保护。就像你装 APP 会看权限一样，装 Skill 也要过安全审计。

给 OpenClaw 用户的建议

如果你也在用 OpenClaw，想安装社区 Skill，我的建议是：

  1. 先装 Skill Vetter — 这是你的"安全卫士"
  2. 不要跳过审计 — 再小的技能也要检查
  3. 关注权限申请 — 警惕申请过多权限的技能
  4. 优先选已 vetted 的 — 社区有"已审计"标签的技能更可靠
  5. 发现问题及时报告 — 保护自己也保护社区

安全不是一个人的事，是整个社区的事。
