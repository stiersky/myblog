📅 2026 年 3 月 19 日 ⚡ 性能优化
💡 问题：MacBook Pro 17GB 内存，运行 Ollama 9b 模型（6.6GB）+ Chrome + 飞书，内存只剩 81MB，系统严重卡顿。

📊 内存占用分析

总内存：17 GB 已用内存：15 GB (88%) 可用内存：81 MB 🔴 压缩内存：4 GB 系统空闲率：18%

Top 占用进程

  • Lark/飞书：~736 MB（主进程 + 渲染进程）
  • Chrome：~1.5 GB+（多标签页）
  • Ollama 模型：6.6 GB（磁盘）/ ~8 GB（运行时）
  • openclaw-gateway：283 MB

🔧 优化方案

1. 内存监控命令

# 查看内存压力
memory_pressure

# 查看进程内存占用
ps aux | sort -nrk4 | head -15

# 查看 Ollama 模型
ollama list

2. 快速释放内存

🚀 立竿见影的方法

  • 关闭 Chrome 不用的标签页（可释放 500MB-1GB）
  • 重启飞书（如暂时不用）
  • 重启 Ollama 服务：`ollama serve` 重启

3. 长期优化策略

# 1. 使用轻量级模型
ollama pull qwen3.5:3b  # 约 2GB

# 2. 限制 Ollama 并发
OLLAMA_NUM_PARALLEL=1 ollama serve

# 3. 定期清理系统缓存
sudo purge  # 需谨慎使用

⚠️ 遇到的坑

坑 1：模型加载慢

首次加载 9b 模型需要 30-60 秒。解决：保持 Ollama 服务常驻。

坑 2：内存泄漏

长时间运行后内存占用持续增长。解决：定期重启服务。

坑 3：交换频繁

内存不足时频繁使用 swap，导致系统卡顿。解决：关闭不必要的应用。

✅ 最佳实践

📋 日常维护清单：
  • 每天检查内存占用：`memory_pressure`
  • 每周清理 Chrome 标签页
  • 每月重启 Ollama 服务
  • 监控系统空闲率，保持 >20%

📊 优化效果

优化后预期效果：

  • 可用内存：81 MB → 2-3 GB
  • 系统空闲率：18% → 30%+
  • 响应速度：提升 50%+

🎓 总结

核心原则：监控 > 优化。定期查看内存状态，及时释放不必要占用。
