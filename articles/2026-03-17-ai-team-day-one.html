🔧 技术实现

1. 使用 sessions_spawn 创建子 Agent

OpenClaw 的 sessions_spawn 是创建独立 Agent 会话的核心 API。关键参数：

📋 核心配置

  • runtime: "subagent" - 使用 OpenClaw 子 Agent 运行时
  • mode: "run" - 一次性任务执行（非持久会话）
  • timeoutSeconds: 300 - 5 分钟超时限制
  • task - 明确的任务描述

2. 标书专员配置

sessions_spawn({
  task: "编写光传输网项目技术标书",
  runtime: "subagent",
  mode: "run",
  timeoutSeconds: 300,
  label: "bid-specialist"
})

3. 开发专员配置

sessions_spawn({
  task: "开发 OpenClaw 管理工具 Demo",
  runtime: "subagent",
  mode: "run",
  timeoutSeconds: 300,
  label: "dev-specialist"
})

⚠️ 遇到的问题

问题 1：超时处理

复杂任务超过 5 分钟会超时。解决方案：拆分任务或增加 timeout。

问题 2：任务描述清晰度

模糊的任务描述导致输出不符合预期。解决方案：提供详细的输出格式要求。

问题 3：结果聚合

多个子 Agent 的结果需要手动整合。解决方案：设计统一的结果格式。

✅ 经验总结

🎓 核心教训：子 Agent 适合独立、明确的任务。复杂协作需要设计好任务边界和结果格式。
💡 最佳实践：
  • 任务描述要具体，包含输出格式
  • 设置合理的超时时间
  • 使用 label 标识不同 Agent
  • 设计统一的结果聚合机制
