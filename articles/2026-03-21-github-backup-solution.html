📅 2026 年 3 月 21 日 📦 GitHub
💡 需求：每天自动备份 workspace 到 GitHub，无需手动输入密码，安全认证。

🎯 认证方案对比

🔐 三种认证方式

  • SSH Key：传统方案，需管理密钥对
  • Personal Access Token：需存储在脚本或环境变量
  • macOS Keychain：✅ 系统级安全管理，推荐使用

🔧 配置步骤

1. 配置 Git 使用 Keychain

# 启用 macOS Keychain 凭证存储
git config --global credential.helper osxkeychain

# 验证配置
git config --global --list | grep credential

2. 首次 Push 触发认证

cd /users/stier/.openclaw/workspace
git add .
git commit -m "Initial commit"
git push origin main
# 首次会弹出 macOS 钥匙串认证对话框
# 输入 GitHub 用户名和 Personal Access Token
# 勾选"保存到钥匙串"

3. 创建备份脚本

#!/bin/bash # github-backup.sh set -e WORKSPACE="/users/stier/.openclaw/workspace" LOG_FILE="$WORKSPACE/logs/github-backup.log" TIMESTAMP=$(date '+%Y-%m-%d %H:%M:%S') mkdir -p "$(dirname "$LOG_FILE")" echo "[$TIMESTAMP] 开始备份..." >> "$LOG_FILE" cd "$WORKSPACE" # 暂存所有更改 git add -A # 检查是否有更改 if git diff --cached --quiet; then echo "[$TIMESTAMP] 无更改，跳过" >> "$LOG_FILE" else # 提交 git commit -m "🤖 自动备份 $(date '+%Y-%m-%d')" echo "[$TIMESTAMP] 已提交" >> "$LOG_FILE" fi # 推送（自动使用 Keychain 凭证） git push origin HEAD 2>&1 | tee -a "$LOG_FILE" echo "[$TIMESTAMP] 备份完成" >> "$LOG_FILE"

4. 配置 launchd 定时任务

# 创建 plist 文件
~/Library/LaunchAgents/com.stiersky.github-backup.plist

# 加载任务
launchctl load ~/Library/LaunchAgents/com.stiersky.github-backup.plist

# 验证状态
launchctl list | grep stiersky

⚠️ 遇到的坑

坑 1：crontab 权限问题

macOS 系统权限限制，crontab 配置失败。解决：改用 launchd。

坑 2：Keychain 未保存凭证

首次 Push 未勾选"保存到钥匙串"。解决：删除凭证重新认证。

坑 3：脚本路径问题

launchd 执行时找不到脚本。解决：使用绝对路径。

✅ 最佳实践

📋 安全清单：
  • 使用 macOS Keychain 管理凭证，不硬编码密码
  • Personal Access Token 设置最小权限（repo 即可）
  • 定期轮换 Token（建议每 90 天）
  • 日志文件不包含敏感信息
  • 脚本设置执行权限：chmod +x script.sh

📊 完整配置汇总

  • 仓库地址：github.com/stiersky/manager.git
  • 执行时间：每天 22:00
  • 认证方式：macOS Keychain
  • 日志位置：workspace/logs/github-backup.log

🎓 总结

核心优势：macOS Keychain + launchd 是 macOS 原生方案，安全可靠，无需额外依赖。
