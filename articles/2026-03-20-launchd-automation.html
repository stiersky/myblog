💡 问题：macOS 系统权限限制，crontab 配置失败（Operation not permitted）。需要系统级定时任务方案。

🎯 为什么选择 launchd

特性 crontab launchd
系统支持 ⚠️ 可能受限 ✅ macOS 原生
权限要求 ⚠️ 需要辅助功能 ✅ 用户级即可
日志管理 ❌ 需手动配置 ✅ 内置支持
错误处理 ❌ 基础 ✅ 完善
学习曲线 ✅ 简单 ⚠️ 中等

🔧 实战配置

1. 创建 plist 配置文件

<?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"> <plist version="1.0"> <dict> <key>Label</key> <string>com.stiersky.github-backup</string> <key>ProgramArguments</key> <array> <string>/bin/bash</string> <string>/users/stier/.openclaw/workspace/scripts/github-backup.sh</string> </array> <key>StartCalendarInterval</key> <dict> <key>Hour</key> <integer>22</integer> <key>Minute</key> <integer>0</integer> </dict> <key>WorkingDirectory</key> <string>/users/stier/.openclaw/workspace</string> </dict> </plist>

2. 放置配置文件

# 用户级任务
~/Library/LaunchAgents/com.stiersky.xxx.plist

# 系统级任务（需要 sudo）
/Library/LaunchDaemons/com.stiersky.xxx.plist

3. 加载任务

# 加载任务
launchctl load ~/Library/LaunchAgents/com.stiersky.github-backup.plist

# 验证状态
launchctl list | grep stiersky

# 卸载任务
launchctl unload ~/Library/LaunchAgents/com.stiersky.github-backup.plist

📋 完整示例：GitHub 备份

备份脚本

#!/bin/bash # github-backup.sh set -e cd /users/stier/.openclaw/workspace git add -A git commit -m "🤖 自动备份 $(date '+%Y-%m-%d')" git push origin HEAD

plist 配置

<?xml version="1.0" encoding="UTF-8"?> <plist version="1.0"> <dict> <key>Label</key> <string>com.stiersky.github-backup</string> <key>ProgramArguments</key> <array> <string>/bin/bash</string> <string>/users/stier/.openclaw/workspace/scripts/github-backup.sh</string> </array> <key>StandardOutPath</key> <string>/users/stier/.openclaw/workspace/logs/github-backup.log</string> <key>StandardErrorPath</key> <string>/users/stier/.openclaw/workspace/logs/github-backup-error.log</string> <key>StartCalendarInterval</key> <dict> <key>Hour</key> <integer>22</integer> <key>Minute</key> <integer>0</integer> </dict> </dict> </plist>

⚠️ 遇到的坑

💣 坑 1：脚本没有执行权限。解决：chmod +x script.sh
💣 坑 2：路径使用相对路径导致失败。解决：使用绝对路径 + WorkingDirectory
💣 坑 3：日志文件目录不存在。解决：脚本开头创建目录 mkdir -p

✅ 最佳实践

📋 配置清单

  • 使用绝对路径，避免相对路径问题
  • 配置 StandardOutPath 和 StandardErrorPath
  • 脚本开头设置 set -e 错误退出
  • 定期清理日志文件，避免占用过大
  • 使用 launchctl list 验证任务状态

🎓 总结

核心优势：launchd 是 macOS 原生方案，权限要求低，日志管理完善，适合自动化任务。
