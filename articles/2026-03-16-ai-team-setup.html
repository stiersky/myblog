🔧 独立 Agent 配置

1. 创建飞书机器人

每个 AI 员工一个独立的飞书机器人应用：

AI 员工 机器人名称 App ID
📝 标书专员 标书专员 cli_a922eca297f85cc8
💻 开发专员 开发助手 cli_a93c05a41d78dcee
🦞 大总管 默认 cli_a923372934b89cd2

2. 配置多 Account

修改 ~/.openclaw/openclaw.json：

{
  "channels": {
    "feishu": {
      "enabled": true,
      "domain": "feishu",
      "accounts": {
        "default": {
          "appId": "cli_a923372934b89cd2",
          "appSecret": "..."
        },
        "bid-account": {
          "appId": "cli_a922eca297f85cc8",
          "appSecret": "..."
        },
        "dev-account": {
          "appId": "cli_a93c05a41d78dcee",
          "appSecret": "..."
        }
      }
    }
  },
  "bindings": [
    {
      "agentId": "main",
      "match": { "channel": "feishu", "accountId": "default" }
    },
    {
      "agentId": "bid-specialist",
      "match": { "channel": "feishu", "accountId": "bid-account" }
    },
    {
      "agentId": "dev-specialist",
      "match": { "channel": "feishu", "accountId": "dev-account" }
    }
  ]
}
💡 关键点：OpenClaw 支持 accounts 嵌套配置，不是独立 channel。每个 account 对应一个飞书机器人应用。

📋 AI 员工规范

每个 AI 员工都有详细的 AGENTS.md，包含：

  • Identity & Personality - 人格特征和沟通风格
  • Core Mission - 核心使命和关键职责
  • Critical Rules - 铁律（不做模糊要求）
  • Technical Deliverables - 标准交付物和质量标准
  • Workflow Process - 详细工作流程
  • Success Metrics - 量化成功指标
  • Communication Protocol - 协作协议

标书专员示例

# ❌ Critical Rules（铁律）

### 标书规范铁律
1. **少表格原则** - ≤4 个表格，多用段落叙述
2. **图表标准** - 10 张专业图表（架构图、流程图等）
3. **页数要求** - 180-200 页，内容详实
4. **响应完整** - 招标文件每条要求必须响应

### 时间铁律
5. **24 小时缓冲** - 投标截止前 24 小时完成初稿
6. **分段交付** - 大项目分章节交付
7. **风险预警** - 发现延期立即汇报

🎁 引入 145 个专业 Agent

安装 agency-agents 仓库，获得 145 个专业 AI Agent 模板：

cd ~/.openclaw/workspace
git clone https://github.com/stiersky/agency-agents.git
cd agency-agents
./scripts/convert.sh --tool openclaw
./scripts/install.sh --tool openclaw
openclaw gateway restart

⚠️ 经验教训

✅ 做对的

  • 独立 Agent 配置 - 每个 AI 员工有独立身份和飞书机器人
  • 详细规范 - AGENTS.md 包含人格、职责、铁律、指标
  • 量化标准 - 成功指标可测量（如响应完整率 100%）
  • 借鉴专业 - agency-agents 提供 145 个模板
⚠️ 踩过的坑：
  • 群聊权限 - 需要配置 groupAllowFrom，否则群消息被丢弃
  • 多 Account 配置 - OpenClaw 支持 accounts 嵌套，不是独立 channel
  • Gateway 重启 - 配置修改后必须重启 Gateway
