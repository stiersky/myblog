🔧 P0 优化：立竿见影（已完成）

1. 心跳本地化

心跳任务每 30 分钟执行一次，全天 48 次。改用本地模型后成本直接归零。

# 配置前：使用云端模型
心跳间隔：30 分钟
模型：qwencode/qwen3.5-plus
单次成本：~¥0.15
月度成本：48 × 30 × ¥0.15 = ¥216

# 配置后：使用本地模型
模型：ollama/myqwen3-fullgpu
单次成本：¥0
月度成本：¥0
⚠️ 注意事项： 本地模型不支持 tools，心跳任务如需调用工具（检查 Gateway、Telegram 等）需改用云端模型。

2. RAG 检索优化

Memory 检索时限制返回片段数量和字符数，减少 70-90% token 消耗。

# RAG 配置
topK: 5          # 最多返回 5 个片段
maxChars: 2000   # 最多 2000 字符
# 优化前：平均 8000 字符 → 优化后：2000 字符
# 节省：75% token

🔧 P1 优化：精细化运营（已完成）

1. 大文件自动摘要

超过 10KB 的文件自动摘要为 200-300 字，避免长文本直接送入模型。

场景 优化前 优化后 节省
10KB 文档 ~5000 tokens ~300 tokens 94%
50KB 文档 ~25000 tokens ~400 tokens 98%

2. 模型选择策略

# 默认策略
default: ollama/myqwen3-fullgpu  # 本地，¥0

# 复杂任务手动指定
complex: qwencode/qwen3.5-plus   # 云端，支持 tools

# 使用原则
1. 简单问答 → 本地模型
2. 需要 tools → 云端模型
3. 创意/分析 → 云端模型

🔧 P2 优化：进阶技巧（可选）

预计额外节省 ¥530/月，适合高频用户：

  1. 会话压缩 - 每 20 轮对话压缩一次，保留关键结论
  2. 上下文分层 - 热/温/冷数据分层管理
  3. 批量处理 - 相似任务聚类执行，减少 API 调用
  4. 缓存机制 - 内存/磁盘/语义多层缓存
